

# План: Полноценная админ-панель с базой данных

Сейчас админ-панель работает на mock-данных в памяти — ничего не сохраняется. Нужно перевести всё на базу данных и добавить новые разделы.

---

## Что будет сделано

### 1. База данных — создание таблиц

Миграция создаст следующие таблицы:

- **services** — услуги (название, описание, цена, длительность, категория, фото, порядок сортировки)
- **masters** — мастера (имя, роль, специализации, опыт, фото, описание)
- **appointments** — записи клиентов (имя, телефон, услуга, мастер, дата, время, статус, комментарий)
- **promotions** — акции (заголовок, описание, фото, скидка, даты начала/окончания, активность)
- **salon_settings** — настройки салона (ключ-значение для адреса, телефона, графика работы и т.д.)

Storage-бакет `salon-images` для загрузки фотографий услуг, мастеров и акций.

RLS-политики: чтение для всех (публичный сайт), запись только для авторизованных админов.

### 2. Аутентификация админа

- Страница `/admin/login` с email + пароль
- Таблица `user_roles` с ролью `admin`
- Защита всех админ-маршрутов — без авторизации редирект на логин
- Google OAuth для удобного входа

### 3. Новая админ-панель — 5 разделов с боковым меню

**Заявки (записи)**
- Таблица всех записей с фильтрами по статусу, дате, мастеру
- Смена статуса: новая → подтверждена → выполнена / отменена
- Поиск по имени/телефону

**Услуги**
- CRUD: добавление, редактирование, удаление услуг
- Загрузка фото услуги с компьютера (в storage)
- Выбор категории, цена, длительность, описание

**Мастера**
- CRUD: добавление, редактирование, удаление мастеров
- Загрузка фото мастера
- Специализации, опыт, описание

**Акции**
- CRUD: создание акций с заголовком, описанием, размером скидки
- Загрузка баннера акции
- Даты проведения, переключатель активности

**Настройки салона**
- Редактирование адреса, телефона, email, графика работы
- Текст «О салоне», «Об оборудовании»

### 4. Обновление публичной части сайта

- Страницы Services, Index, About, BookingModal переключатся на данные из БД вместо статического файла `data/services.ts`
- Новый блок «Акции» на главной странице
- Фотографии мастеров и услуг подтягиваются из storage

### 5. Заполнение начальными данными

Текущие данные из `data/services.ts` (17 услуг, 5 мастеров) будут вставлены в БД как начальные данные.

---

## Технические детали

- **Sidebar**: Shadcn Sidebar с иконками (Calendar, Scissors, Users, Sparkles, Settings)
- **Формы**: shadcn Input, Select, Textarea + загрузка файлов через `<input type="file">`
- **Запросы**: React Query + Supabase SDK для всех CRUD-операций
- **Storage**: публичный бакет `salon-images` с папками `services/`, `masters/`, `promotions/`
- **Роутинг**: `/admin/login`, `/admin` (с вложенными разделами через табы или sidebar)
