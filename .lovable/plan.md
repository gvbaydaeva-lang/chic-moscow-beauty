

# План: AI-чат-бот с записью на услуги

## Что будет сделано

### 1. База данных — новые таблицы

**chat_sessions** — сессии чата:
- `id`, `client_name`, `client_phone`, `status` (active / operator_requested / closed), `created_at`, `updated_at`
- RLS: публичная вставка, админы видят все, клиент по ID из localStorage

**chat_messages** — сообщения:
- `id`, `session_id` (FK → chat_sessions), `role` (user / assistant / operator), `content`, `created_at`
- RLS: аналогично chat_sessions

Realtime включён для обеих таблиц (для админки и клиентского чата).

### 2. Edge Function — AI-ассистент (`chat-ai`)

- Принимает `session_id` + новое сообщение пользователя
- Загружает из БД: все услуги, мастеров, историю сообщений
- Формирует system prompt с контекстом салона TERRA (услуги, цены, мастера)
- Вызывает Lovable AI (gemini-2.5-flash) для генерации ответа
- Если пользователь подтверждает запись — создаёт заявку в таблице `appointments`
- Если ИИ не может помочь — меняет статус сессии на `operator_requested`
- Сохраняет ответ в `chat_messages`

### 3. Компонент чат-виджета (`ChatWidget`)

Плавающая кнопка в правом нижнем углу на всех страницах:

1. **Шаг 1 — Регистрация**: имя, телефон, чекбокс «согласен с политикой конфиденциальности»
2. **Шаг 2 — Чат**: интерфейс переписки с ИИ-ассистентом
   - Сообщения пользователя и бота
   - Поле ввода + кнопка отправки
   - Кнопка «Позвать оператора»
   - При оформлении записи — подтверждение в чате

Виджет добавляется в `Layout.tsx`, виден на всех публичных страницах.

### 4. Админка — раздел «Чаты»

Новый пункт в сайдбаре админки (иконка MessageSquare):

- Список всех чат-сессий с фильтром по статусу
- Бейдж с количеством тикетов (operator_requested)
- При клике — просмотр всей переписки
- Возможность ответить от имени оператора (role: operator)
- Кнопка закрыть/вернуть чат

### 5. Роутинг

- Добавить `/admin/chats` → `AdminChats.tsx`
- Добавить `AdminChatsRoute.tsx` с AdminLayout-обёрткой

---

## Технические детали

- **AI модель**: `google/gemini-2.5-flash` через Lovable AI Gateway
- **System prompt**: включает полный каталог услуг/мастеров из БД, инструкцию помогать с выбором и оформлять запись
- **Realtime**: Supabase Realtime для обновления чата в реальном времени (и у клиента, и в админке)
- **Storage сессии**: `session_id` хранится в `localStorage` клиента для продолжения чата
- **Markdown**: ответы бота рендерятся через `react-markdown`
